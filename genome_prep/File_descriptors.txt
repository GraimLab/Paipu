This directory is where genomes for a desired mammal are queried from NCBI to identify which reference genome to use for that mammal and to prep it for use in FREYA.

- This directory has been cleaned from previous attempts to utilize TOGA genomes (which failed due to inconsistency in chromosome names and qualities) and create a pipeline for processing all Zoonomia genomes. This task became too hard, as querying genomes for all animals often resulted in no reference genome, then subseqency checks for other possible genomes and TOGA matches created too large many decisions which often resulted in having no genome anyway.  
- Now, only for a specified list of mammals that we actually have RNA-Seq data for, we query the mammal with the following pipeline:
	1) Query identified reference genome on NCBI.
	2) If reference available on NCBI, take it and prep that genome using Snakemake workflow
	   Else manually inspect - species may another genome available (even if not reference) if not, inspect TOGA genome (make sure chromosomes match).



------------- Script/File Descriptions -------------
-- FILES --
 
- organism_row_counts_Bria.csv : original list of mammals to be queried from Bria and the number of samples we have for each

- ALL.csv : NCBI query results for mammals in organism_row_counts_Bria.csv. 
Columns in this file:
SPECIES,ACCESSION,ASSEMBLY_NAME,ASSEMBLY_STATUS,RELEASE_DATE,HAS_ANNOATATION,ANNOTATION_PROVIDER,ANNOTATION_RELEASE_DATE

- ALL_GCA_only.csv : Subsequent minor investigations of the ALL.csv file. This is looking at mammals that only have GCA (versus GCF) genomes, these often do not have gene annotations associated with them and are not identified as 'current/reference' by NCBI.

- ALL_no_quotes.csv : Subsequent minor processing of the ALL.csv file. This gets rid of quotes within ALL.csv.

- config_test.txt : Example of what a prepped config file for a mammal for FREYA looks like.  

- input.txt : Trimmed list of organism_row_counts_Bria.csv getting rid of the sample counts so that we can query from this file. 

- commands_to_fix_mistake.txt : I must have messed something up when updating some version info in the config file. I'm not quite sure what happened but this is a record of commands used to fix whatever mistake I made (???). Scared to delete - though this is not part of the main pipeline. 

- mammal_names_only.txt : Same as input.txt, but replace spaces with underscore. This was likely done for testing for the json query.

- ncbi_queries/temp.json : example of entire query from NCBI (we only take the recommended query from this output).

- RefSeq_genomes.csv : From queried genomes, get the genomes that have a 'RefSeq' genoems (these are often the GCF genomes that have a gene annotation and can be cleared to be further processed).

- Ref_Seq_no_quotes.csv : Same as above - no quotes.


-- SCRIPTS --

- ncbi_queries : Directory that holds all NCBI json queries for mammals as well as associated scripts.

- ncbi_queries/query.sh : Example commands to query mammals from NCBI.

- ncbi_queries/json_parse.sh : Script to parse the output from query.sh.


------------- Example Mammale Query -------------
1) Create input file that has list of mammals to query. 
Example format:

Tupaia chinensis
Bos indicus 


2) run ncbi_queries/query.sh (it expects to read in an input file called input.txt formated as above
This file will load the necessary modules, read in the input.txt file, query genomes for each mammal, parse the query for each mammal (json queries are kept), and write final information for each mammal genome to output.txt.

output.txt can then be used to get the genome for each mammal and run through processing pipeline.

Please note all files associated with querying genomes will be in ncbi_queries.
